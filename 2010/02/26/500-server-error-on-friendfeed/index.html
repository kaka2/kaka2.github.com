2月 26 2010

500 server error on friendfeed

FriendFeed杯具了

500 Internal Server Error

nginx/0.6.34

這是剛才開啟Puff后，再次訪問FrienFeed后反饋的結果，杯具發生了，唉！

意外時時有，但這次，我寧愿沒有經歷過。

下午15:30前后的樣子，想用Gtalk更新FriendFeed和Twitter都沒有成功，起初只是看到機器人ff@chat.friendfeed.com處于下線狀態，當然不能接受我發出去的信息了；接著，通過YR讀到了TechCrunch上的一篇消息，反映了在Twitter上僅僅有50條Tweets在它down機的30分鐘內出現。這不得不令人疑慮，是不是Facebook 收購（2009年8月10日）FF后看到沒有多少用戶使用了，于是就想關閉服務呢？

我一直沒有放棄FriendFeed，她是個挺不錯的社會化網絡服務，提供了豐富的內容聚合，而且是前Google工程師們開發出來的，UI界面很清爽，blogger們對她的“高明之處”亦有不錯的贊譽之言。

用Google.com搜索關鍵詞FriendFeed，可以得到34,200,000條相關條目。
